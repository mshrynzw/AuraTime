# AuraTime ドキュメント一覧

本ディレクトリには、マルチテナント勤怠・給与管理システム「AuraTime」の設計・運用に関するドキュメントが格納されています。

---

## 📖 ドキュメントの読み方

### 開発を始める前に

1. **[概要](./100_基本設計/100_概要.md)** でシステム全体像を把握
   - システム目的、対象ユーザー、スコープを確認

2. **[用語集](./000_要件定義/020_用語集.md)** で用語を統一
   - システム内で使用される用語の定義を確認

3. **[基本設計](./100_基本設計/110_基本設計.md)** で機能一覧と権限モデルを確認
   - 機能要件、画面一覧、権限モデルを理解

4. **[業務フロー](./100_基本設計/150_業務フロー.md)** で業務の流れを理解
   - 勤怠から支払までのプロセス、ロック制御を把握

5. **[開発順序](./300_開発/300_開発順序.md)** で推奨開発順序を確認
   - フェーズごとの実装内容と完了基準を理解

### 実装時

#### UI実装

- **[画面遷移図](./200_詳細設計/220_UI_遷移図.mmd)** で画面遷移を確認
- **[画面レイアウト案](./200_詳細設計/221_UI_レイアウト.mmd)** でレイアウトを確認

#### 機能実装

- **[基本設計](./100_基本設計/110_基本設計.md)** で機能仕様を確認
- **[API基本方針](./200_詳細設計/230_API.md)** でAPI設計方針を確認
- **[API仕様](./200_詳細設計/231_API_勤怠.md)** で各APIの詳細仕様を確認

#### データベース操作

- **[DB設計方針](./200_詳細設計/211_DB.md)** でテーブル概要を確認
- **[ER図](./200_詳細設計/210_ER図.mmd)** でテーブル間の関係を確認
- **[DDL運用](./200_詳細設計/212_DDL運用.md)** でマイグレーション管理を確認

#### セキュリティ・非機能要件

- **[セキュリティ設計](./200_詳細設計/250_セキュリティー.md)** でセキュリティ要件を確認
- **[性能設計](./200_詳細設計/210_性能.md)** でパフォーマンス要件を確認
- **[ログ・監査設計](./200_詳細設計/260_ログ.md)** でログ設計を確認

### テスト・運用時

- **[テスト設計](./400_テスト/400_テスト設計.md)** でテスト方針を確認
- **[運用設計](./900_リリース/900_運用設計.md)** で運用方針を確認

## 📝 ドキュメント更新方針

### 更新タイミング

- **設計変更時**: 該当する詳細設計書を更新
- **仕様変更時**: 必ず関連するドキュメントを更新
- **新機能追加時**: 基本設計書、詳細設計書を更新

### バージョン管理

- 各ドキュメントのヘッダーにバージョン情報を記載（必要に応じて）
- 重要な変更時はバージョンを更新

### レビュー

- 設計変更時は関連ドキュメントの整合性を確認
- 実装完了後、設計書との整合性を確認
