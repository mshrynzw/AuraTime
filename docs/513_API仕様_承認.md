# API仕様：承認

## 1. 承認待ち一覧

### GET `/api/v1/approval-requests`
自身が承認すべきリクエスト一覧を取得。

- **クエリパラメータ**: `status=pending`
- **リクエスト全体のステータス**: `pending`, `approved`, `rejected`, `canceled`
- **権限**: Manager以上

## 2. 承認アクション

### POST `/api/v1/approval-requests/{id}/act`
承認または却下を実行。

- **リクエスト**:
    ```json
    {
      "action": "approve",
      "comment": "確認しました。"
    }
    ```
- **内部処理**:
    - `approval_steps` の現在ステップを更新。
    - 最終ステップであれば、対象リソース（`time_records` 等）のステータスを `approved` に更新。
    - 次の承認者がいる場合は、`current_approver_user_id` を更新。

## 3. 申請詳細

### GET `/api/v1/approval-requests/{id}`
承認ルート（ステップ）の履歴を含む詳細情報を取得。
